# ПОЛНАЯ ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ: FLOVER SHOP ERP/CRM

## 1. Введение
**Flover Shop** — это комплексная информационная система (ERP/CRM), предназначенная для автоматизации рабочих процессов цветочного магазина. Система охватывает управление складскими запасами, клиентской базой, продажами, поставками и финансовыми расходами.

Проект разработан с учетом требований к современным веб-приложениям: масштабируемость, контейнеризация и документируемость через API.

---

## 2. Архитектура системы
Система построена на базе классической клиент-серверной архитектуры:
- **Presentation Layer (Admin Panel):** Серверный рендеринг на базе движка **EJS** (Embedded JavaScript templates), обеспечивающий высокую скорость загрузки и простоту разработки интерфейса.
- **Business Logic Layer (Backend):** RESTful API на базе **Express.js** и Node.js.
- **Data Access Layer (ORM):** Библиотека **Sequelize**, обеспечивающая абстракцию над базой данных.
- **Persistence Layer (Database):** Реляционная СУБД **PostgreSQL**, запущенная в Docker-контейнере.

## 4. Структура базы данных (Schema)
База данных состоит из 11 основных таблиц, связанных между собой сложной системой внешних ключей.

### 4.1. Основные сущности
1. **Products (`product`)**: Хранит информацию о товарах (не только цветах, но и инструментарии, удобрениях).
   - Поля: `id`, `name`, `unit`, `id_category`, `stock_quantity`.
2. **Product Groups (`productGroup`)**: Категории товаров. Поддерживает иерархическую структуру (самореференс).
3. **Customers (`customer`)**: Клиентская база с учетом персональных скидок и истории покупок.
4. **Sales (`sale`)**: Журнал транзакций продаж.
5. **Shipments (`shipment`)**: Учет входящих поставок от поставщиков.
6. **Providers (`provider`)**: Справочник поставщиков.

### 4.2. Связующие таблицы
- **`prod_is_on_sale`**: Связь "Многие-ко-многим" между товарами и продажами (позиции чека).
- **`prod_in_list_price`**: Связывание товаров с прайс-листами.

### 4.3. Отношения (ERD)
- **Product Group**: Имеет связь `belongsTo` сама на себя (`id_base_goods_group`) для создания подкатегорий.
- **Product**: Принадлежит категории.
- **Sale**: Принадлежит клиенту и конкретному прайс-листу.
- **Shipment**: Автоматически обновляет поле `stock_quantity` в таблице `product` при поступлении.

## 5. Бизнес-логика и ключевые функции

### 5.1. Управление инвентарем
Система отслеживает текущие остатки товаров. 
- **Поступление товара:** При создании записи в таблице `shipment`, количество товара в таблице `product` автоматически увеличивается.
- **Продажа товара:** При оформлении продажи через интерфейс, количество соответствующего товара уменьшается на указанный объем.

### 5.2. Ценообразование
Система поддерживает использование различных прайс-листов (например, "Оптовый", "Розничный", "Праздничный"). Продажа всегда привязана к конкретному прайс-листу.

### 5.3. Аналитика расходов
Модуль `expenses` позволяет фиксировать операционные затраты магазина: аренда, логистика, зарплаты и т.д., что позволяет в дальнейшем рассчитывать чистую прибыль.

## 6. Описание интерфейса (Admin Panel)

Интерфейс доступен по адресу: `http://localhost:8080/`

1. **Dashboard (Главная)**: Сводная статистика по продажам и основным показателям.
2. **Инвентарь (`/inventory`)**: Мониторинг остатков. Критические уровни запасов подсвечиваются (например, если цветов осталось мало).
3. **Продажи (`/sales`)**: Журнал всех совершенных сделок. Возможность просмотра деталей каждой продажи.
4. **Клиенты (`/customers`)**: Список покупателей с их телефонами и размером персональной скидки.
5. **Поставки (`/shipments`)**: Окно для регистрации новых поступлений товара. Позволяет выбрать товар и поставщика.
6. **Расходы (`/expenses`)**: Финансовое планирование и учет затрат.

## 7. Установка и запуск (Deployment)

### 7.1. Предварительные требования
- Установленный **Docker** и **Docker Compose**.
- Установленная **Node.js** (для локальной разработки без Docker).

### 7.2. Настройка окружения
В корне проекта должен находиться файл `.env`. Пример заполнения:
```env
# Database Config
POSTGRESDB_USER=postgres
POSTGRESDB_ROOT_PASSWORD=your_password
POSTGRESDB_DATABASE=flover_shop
POSTGRESDB_LOCAL_PORT=5433
POSTGRESDB_DOCKER_PORT=5432

# Node.js App Config
NODE_LOCAL_PORT=8080
NODE_DOCKER_PORT=8080
```

### 7.3. Запуск через Docker (Рекомендуется)
```bash
docker-compose up --build
```
Эта команда поднимет базу данных, создаст таблицы через Sequelize и запустит веб-сервер.

### 7.4. Запуск без Docker
1. Перейдите в папку `flover_shop-app`.
2. Выполните `npm install`.
3. Убедитесь, что у вас есть запущенный экземпляр PostgreSQL.
4. Запустите: `node server.js`.

## 8. API Инструкции

### 8.1. Swagger UI
Интерактивная документация всех эндпоинтов доступна по адресу:
`http://localhost:8080/api-docs/`

### 8.2. Основные точки входа (Endpoints)
- `GET /api/products` — список всех товаров.
- `POST /api/sales` — создание новой продажи (одновременно списывает товар со склада).
- `GET /api/customers` — управление клиентской базой.
- `GET /api/shipments` — просмотр истории поставок.

## 9. Дополнительные утилиты

### Модуль печати чеков (Frontend Utility)
В папке `frontend` находится скрипт `index.js`, предназначенный для эмуляции работы термопринтера.
- **Запуск:**
  `node frontend/index.js [ID_SALE]`
- **Функционал:** Получает данные о продаже по ID из базы данных и выводит в консоль красиво оформленный чек с итоговой суммой и списком позиций.
